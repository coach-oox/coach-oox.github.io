React Hook Form

Dohae Lee on Jan 192022-01-19T11:51:17+08:00
react
14 min read

React Hook Form은 React에서 form을 보다 쉽게 다루기 위한 라이브러리이다. form을 핸들링 하기 위해 작성해야 하는 코드 중에는 일반적으로 중복되는 노가다가 많은데, React Hook Form을 사용한다면 이러한 단점을 보완할 수 있다.

React Hook Form와 비슷한 라이브러리로는 Formik이라는 친구도 있다. NPM Trends를 기준으로 보자면 거의 비슷한 수준의 인지도를 가지고 있는데, 어느 것을 써도 비슷한 효과를 얻을 수 있으나, 내 경우 React Hook Form이 더 젊고, 업데이트가 잘 되서 사용한다.

어쨌든, 이 친구는 우선 우리들의 귀찮음을 줄여준다는 점에서 이미 큰 점수를 얻고 들어가는데, 성능도 꽤 괜찮고, 유효성 검사에도 능하다는 장점까지 갖췄다. 오피셜 소개에도 나와있는 부분이지만, React Hook Form 라이브러리의 장점을 간단히 요약하자면 다음과 같다.

  • 타입스크립트 기본 지원
  • 적은 코드의 양
  • 좋은 퍼포먼스
  • 빠른 마운팅 속도

올인원 패키지

먼저 다음은 사용자로부터 닉네임을 입력받아 제출하고자 하는 (React Hook Form을 사용하지 않은) 간단한 코드 예제 이다.

const [nickname, setNickname] = useState("");

const onChange = (event) => {
  setNickname(event.target.value);
};

const onSubmit = (event) => {
  event.preventDefault();
};

return (
  <div>
    <form onSubmit={onSubmit}>
      <input value={nickname} onChange={onChange} type="text" />
      <button>Save</button>
    </form>
  </div>
);

구성이 단순해서 지금은 나쁘게 보이진 않지만, form을 핸들링 할 때마다 거의 비슷한 코드를 작성해야 한다는 단점이 있다. 예를들어 사용자 이름, 나이, 전화번호, 성별, 직업, 주소를 입력받는 폼을 구성한다고 생각해보자. 각각을 state로 만들어주고, 각각에 핸들러를 만들어 달아주고… 😂

React Hook Form는 위와 같은 문제를 한 방에 해결할 수 있는 올인원 패키지를 제공한다. 본문에서는 간단한 사용법만 다루기 때문에, 보다 자세한 내용이 필요하다면 다음의 링크 (오피셜 문서)를 참고하도록 하자. 다음은 useForm()에서 제공되는 함수들을 사용해 form을 핸들링 하는 기초적인 방법에 대한 내용이다.

  • useForm
  • useController
  • useFormContext
  • useWatch
  • useFormState
  • useFieldArray

import { useForm } from "react-hook-form";

function Nickname() {
  const { register } = useForm();

  return (
    <div>
      <form onSubmit={handleSubmit(onValid)}>
        <input {...register("nickname")} type="text" placeholder="Nickname" />
        <button>Save</button>
      </form>
    </div>
  );
}

useForm()으로 생성된 register() 함수는 name, onBlur(), onChange() 등의 데이터 및 함수를 포함한 객체를 리턴해준다. 쉽게 말해, 우리가 직접 작성해야 했던 useState() 부분과 onChange() 등을 자동으로 생성해준다. 이때 함수의 필수 인자인 첫 번째 (nickname) 텍스트 인자는 핸들링 하는 대상을 구별하는 이름이 된다. (참고 : register)

setValue

위젯의 값을 변경하고 싶을 땐 다음 코드 예제와 같이 setValue()를 사용한다. 이때 인자로는 등록된 위젯의 이름과 변경할 값을 넘긴다. (참고 : setValue)

1
2

const { setValue } = useForm();
setValue("nickname", "");

defaultValue

만약 위젯에 기본 값을 지정하고 싶다면, 다음 코드 예제와 같이 useForm의 인자로 defaultValue를 지정한 객체를 넘긴다.

1

const { register } = useForm({
  defaultValues: {
    nickname: "Something",
  },
});

watch

watch()를 사용하면 register()로 등록된 위젯들의 변화를 (실시간으로) 감시할 수 있다. 이때 한 번에 여러 개의 위젯을 감시하는 경우, 등록된 이름으로 위젯을 구별할 수 있다. (참고 : watch)

1
2
3

const { watch } = useForm();
console.log(watch("nickname")); // 이름이 nickname으로 등록된 위젯만 감시
console.log(watch()); // 모든 위젯 감시

handleSubmit

form의 onSubmit 이벤트를 핸들링하기 위해서는 handleSubmit()을 사용하면 된다. 이 함수는 폼이 성공적으로 제출 되었을 때 실행할 함수 (Required)와, 에러가 발생했을 때 실행할 함수 (Optional)를 인자로 받아, 각각을 상황에 맞게 실행시켜주는 역할을 한다. 이때 에러가 발생했을 때 실행할 함수는 선택적 인자이므로 생략할 수 있다.

function Nickname() {
  const { register, handleSubmit } = useForm();

  const onSubmit = (data, event) => {
    console.log(data);
  };

  const onError = (errors, event) => {
    console.log("Error!");
  };

  return (
    <div>
      <form onSubmit={handleSubmit(onSubmit, onError)}>
        <input {...register("nickname")} type="text" placeholder="Nickname" />
        <button>Save</button>
      </form>
    </div>
  );
}

폼이 성공적으로 제출되었을 때 실행되는 SubmitHandler는 data와 event 2개의 Props를 가지며, 실패했을 때 실행되는 SubmitErrorHandler는 errors와 event 2개의 Props를 가진다. (참고 : handleSubmit)

formState

formState는 폼의 상태 정보를 포함하고 있는 객체이다. (참고 : formState)

1
2

const { formState } = useForm();
console.log(formState.errors);

formState 객체는 isValid, isSubmitting 등의 정보를 가지고 있으며, errors를 사용해서 등록된 위젯들에서 발생하는 에러를 확인할 수 있다. 이때 errors는 단순히 에러 발생 여부만 감시하는 것이 아니라, 어떤 위젯에서 어떤 타입의 에러가 발생했고, 어떤 에러 메시지를 출력하는지 등을 포괄적으로 관리할 수 있다. (에러 메시지 설정은 하단의 ‘유효성 검사 조건 추가’ 항목을 참고)

에러 메시지 처리

formState.errors 객체를 사용하면 어떤 에러 메시지가 발생했을 때, 그게 required 에러인지 minLength 에러인지 구분할 필요없이, 간단하게 에러 메시지를 핸들링할 수 있다.

1

console.log(formState.errors.nickname.message);

즉, nickname이라는 하나의 위젯에, 체크해야 할 에러가 2개든, 5개든, 10개든지간에, formState.errors.nickname 객체 하나로 다룰 수 있다.

유효성 검사 조건 추가

React Hook Form은 HTML 표준 유효성 검사를 기반으로, 편리하게 유효성 검사 조건을 추가할 수 있게끔, 다음과 같은 옵션을 지원한다. (참고 : Apply validation)

  • required
  • min
  • max
  • minLength
  • maxLength
  • pattern
  • validate

return (
  <div>
    <form onSubmit={handleSubmit(onSubmit, onError)}>
      <input
        {...register("nickname", {
          required: true,
          minLength: 5,
          maxLength: 10,
        })}
        type="text"
        placeholder="Nickname"
      />

      <button>Save</button>
    </form>
  </div>
);

추가할 옵션은 register()의 두 번째 인자에 객체로 작성한다.

에러 메시지 추가

formState.errors 객체를 통해 살펴보면, 에러가 발생할 때 해당 객체에 message라는 항목이 있는 것을 확인할 수 있는데, 이를 옵션을 추가하는 방식으로 간단하게 지정할 수 있다. 예를들어 required 필드는 true 또는 false 값을 가지지만, 여기에 텍스트를 작성하면 required를 만족하지 못했을 경우 (에러 발생시) 출력되는 에러 메시지가 된다.

사용자 정의 에러 핸들링

에러를 (유효성 검사) 옵션 형태로 지정하는 것은 편리하나, 실무에서는 이 기능만으로 원하는 에러를 모두 핸들링 하기에는 부족한 감이 없잖아 있다. 예를들어서 사용자가 제출한 데이터가 데이터베이스에 이미 있는 데이터라던지. 이런 경우에는 사용자가 별도로 에러를 핸들링 할 수도 있다.

validate

다음 코드 예제는 register()에 validate 옵션을 통해 에러를 핸들링 하고 있다. validate는 함수를 가지는데, 이 함수는 현재 값을 의미하는 value를 인자로 가진다. 즉, 이 함수 내부에서 현재 값에 대한 어떤 조건을 검사하고 true 또는 false를 리턴해 에러를 발생 시킬 수 있다. (true를 리턴하면 유효성 검사 통과, false를 리턴하면 에러 발생)

1

<input
  {...register("nickname", {
    validate: {
      checkHello: (value) =>
        !value.includes("hello") || "Hello is already using.",
    },
  })}
  type="text"
  placeholder="Nickname"
/>

이때 문자열을 리턴하면 에러 메시지로 전달할 수 있어, 위 코드 예제와 같이 참인 조건을 만족하지 못할 경우 false가 아닌 에러 메시지를 바로 전달할 수도 있다.

setError

다음 코드 예제는 useForm()이 제공하는 setError() 함수를 사용해서, 폼을 제출할 때 특정 조건에 따라 에러를 발생 시키고 있다. setError()의 첫 번째 인자로는 등록된 이름을 넘기고, 두 번째 인자를 통해서는 에러 메시지를 전달할 수 있다.

1

const { setError } = useForm();

const onSubmit = (data) => {
  if (data.nickname === "hello") {
    setError("nickname", { message: "Hello is already using." });
  }
};
