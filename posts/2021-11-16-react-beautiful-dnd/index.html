react-beautiful-dnd 기본 사용법 총정리

Dohae Lee on Nov 16, 20212021-11-16T00:00:00+08:00
react
8 min read

react-beautiful-dnd는 React에서 Drag & Drop UI를 보다 쉽게 만들 수 있게 도와주는 라이브러로, 비슷한 라이브러리인 react-dnd나 react-draggable에 비해 비교적 단순한 UI 제작에 적합한 친구다.

인기도는 NPM Trends를 기준으로 react-dnd와 비슷한 수준이며, 개인적으로 자유도는 react-dnd가 높고, 단순한 UI에 모던한 느낌 (애니메이션 기본 적용)을 주고 싶다면 react-beautiful-dnd가 훨씬 난이도는 낮은 것 같다. 만들고자하는 UI에 따라 결정하면 될 듯.

DragDropContext

  • DragDropContext

가장 먼저 DND를 구성할 애플리케이션 (DND를 구성할 영역)을 DragDropContext 컴포넌트로 감싼다. 이때 사용자가 드래그를 끝낸 시점에 호출되는 onDragEnd라는 (함수) Prop이 요구되며, 하위에 자식 컴포넌트 (Droppable)를 배치해야 한다.

import { DragDropContext } from "react-beautiful-dnd";

function App() {
  const onDragEnd = (event) => {
    console.log(event);
  };

  return <DragDropContext onDragEnd={onDragEnd}></DragDropContext>;
}

onDragEnd

onDragEnd가 제공하는 인자 (위 코드 예제에서 event)는 source (드래그를 시작한 지점에 관한 정보), destination (드롭한 도착지에 관한 정보) 등 드래그와 드롭에 대한 정보를 담고 있다. 따라서, 이 객체가 제공하는 정보를 토대로, 드래그가 끝날 시점에 처리할 액션을 지정할 수 있다. (e.g., 드롭하면 해당 위치로 변경 등)

Droppable

  • Droppable

그 다음엔 DND를 사용할 수 있는 영역 (한 마디로 드래그 해와서 붙여넣을 수 있는 공간)을 Droppable 컴포넌트로 감싼다. 이때 각 영역을 구분하는 droppableId라는 Prop이 요구되며, 하위에 React 요소가 아닌, 함수 형태의 자식을 배치해야 한다. (이 함수는 ReactElement를 리턴한다.)

import { DragDropContext, Droppable } from "react-beautiful-dnd";

function App() {
  const onDragEnd = () => {};
  return (
    <DragDropContext onDragEnd={onDragEnd}>
      <Droppable droppableId="drag">
        {(provided) => (
          <div ref={provided.innerRef} {...provided.droppableProps}>
            {provided.placeholder}
          </div>
        )}
      </Droppable>
    </DragDropContext>
  );
}

Droppable 하위에 요구되는 함수는 provided, snapshot이라는 2개의 인자를 제공한다.

provided

provided는 innerRef, droppableProps, placeholder 3가지를 제공하는데, 이때 innerRef는 해당 요소를 특정 (지정)하고 조작하기 위해서 반드시 요구되는 부분으로, useRef()와 비슷한 역할을 한다. (참고 : Using innerRef) 위 코드 예제와 같이 droppableProps에서 제공하는 것들을 풀어놓음으로써 해당 요소를 Drop-able하게 만들 수 있다.

snapshot

snapshot에서 제공하는 속성은 다음과 같다. (참고 : snapshot)

type DroppableStateSnapshot = {|
  isDraggingOver: boolean,
  // 현재 영역이 Dragging-Over 되었는지 (끌어다 올린 상태인지) 여부를 boolean 값으로 리턴
  // 즉, isDraggingOver ? 현재 마우스 오버 된 영역 : 다른 영역
  draggingOverWith: ?DraggableId,
  // 현재 Dragging-Over된 DraggableId를 리턴
  // 즉, Boolean(draggingOverWith) ? 현재 마우스 오버 된 영역과 동일한 DraggableId 값을 가진 개체 : 아닌 개체
  draggingFromThisWith: ?DraggableId,
  // 현재 Dragging 중인 개체의 DraggableId를 리턴
  // 즉, Boolean(draggingFromThisWith) ? 현재 드래그 중인 개체와 동일한 DraggableId 값을 가진 개체 : 아닌 개체
  isUsingPlaceholder: boolean,
|};

const Box = styled.div`
  background-color: ${
    (props) =>
      props.isDraggingOver // Dragging-Over 된 영역인가?
        ? "lightpink" // Dragging-Over 되었다면 배경색은 lightpink
        : props.draggingFromThisWith // Dragging-Over 되지 않았다면, 드래그가 시작된 영역인가?
        ? "lightgrey" // 드래그가 시작된 영역이라면 배경색은 lightgrey
        : "lightblue" // 이도 저도 아니라면 배경색은 lightblue
  };
`;

Draggable

  • Draggable

마지막으로 실제 사용자가 마우스로 잡고 드래그 하는 영역을 Draggable로 만든다. Draggable는 draggableId, index라는 Props가 요구되며, Droppable과 마찬가지로 함수 형태의 자식을 하위에 배치한다.

import { DragDropContext, Droppable, Draggable } from "react-beautiful-dnd";

function App() {
  const onDragEnd = () => {};
  return (
    <DragDropContext onDragEnd={onDragEnd}>
      <Droppable droppableId="drag">
        {(provided) => (
          <div ref={provided.innerRef} {...provided.droppableProps}>
            <Draggable draggableId="hello" index={0} key="hello">
              {(provided) => (
                <div ref={provided.innerRef} {...provided.draggableProps}>
                  <span {...provided.dragHandleProps}>Click</span>
                  Hello
                </div>
              )}
            </Draggable>
            {provided.placeholder}
          </div>
        )}
      </Droppable>
    </DragDropContext>
  );
}

참고로 정상적으로 Sorting 하려면 key와 draggableId을 동일하게 지정해야 한다.

provided

Draggable 역시 provided와 snapshot을 제공한다. Draggable에서 제공되는 provided는 innerRef, draggableProps, dragHandleProps 3가지를 제공하며, innerRef는 Droppable과 마찬가지로 필수적으로 요구되는 부분이며, draggableProps은 innerRef와 동일한 노드에 적용되어야 한다.

dragHandleProps의 경우, 실질적으로 DND 기능을 사용할 때 클릭할 수 있는 부분. 즉, 드래그 할 수 있는 부분에 해당 속성을 추가하면 된다. (모두 하나의 요소에 함께 사용할 수도 있다.) 따라서, 위 코드 예제에서는 li 요소 전체가 드래그 가능한 요소가 되지만, 사용자가 드래그 할 수 있는 부분은 span 요소로 한정된다.

snapshot

snapshot에서 제공하는 속성은 다음과 같다. (참고 : snapshot Droppable에서 제공되는 snapshot은 Droppable로 감싸진 (드래그가 가능한) 전체 영역에 관한 Props였다면, Draggable에서 제공되는 snapshot은 각각의 (드래그가 가능한) 아이템들에 관한 Props가 제공되며, 일반적으로 isDragging만 사용된다.

type DraggableStateSnapshot = {|
  // Generally this is the only property you will be using
  isDragging: boolean, // 현재 Dragging 중인지
  isDropAnimating: boolean,
  dropAnimation: ?DropAnimation
  draggingOver: ?DroppableId,
  combineWith: ?DraggableId,
  combineTargetFor: ?DraggableId,
  mode: ?MovementMode,
|};
