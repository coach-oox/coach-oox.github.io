styled-components 기본 사용법 요약 정리

Dohae Lee on Nov 29, 20212021-11-29T00:00:00+08:00
react css-in-js
8 min read

멘토링 하면서 느낀점 푸념 한 줄 : 제발… styled-components 먼저 배우고 (잠깐이라도 보고) emotion 써주세요… 물론 그냥도 잘하는 친구들도 많지만, 가끔 잘못 이해하고 있는 (도대체 여태 어떻게 써온건지, 상상의 나래로 이해해서 틀렸지만 굴러가는 신기한 케이스들이 있음) 친구들 바로 잡아주려면 거의 처음부터 다시 해야 합니다… CSS 좀 하라고 그러면 ‘엥? 전 개발자 될건데요? 전 디자인 개뿔 시른대요?’ 하지 마시고 제발 CSS 배워주세요…

어쨌든 그래서 적어보는 styled-components 입문자를 위한 요약 정리 글. 우선 CSS는 당연히 할 줄 안다는 가정하에 작성했고, 설치 커맨드나 이런 자질구레한건 할 줄 안다는 가정하에 딱 styled-components를 제대로 활용하기 위한 것들만 요약했다. styled-components와 emotion의 차이점에 대해서는 좀 오래된 글이긴 하지만 전에 썼던 Emotion 특징 정리 및 Styled-Components와의 비교를 참고하자. (언젠가 큰 변화가 생기면 리뉴얼 할 예정)

글로벌 스타일 지정

글로벌 스타일을 지정할 때는 createGlobalStyle을 사용한다. 해당 내용은 이전에 작성한 React Styled-Components 글로벌 공통 스타일 지정하기 링크로 대신한다.

다른 HTML 요소로 변경

같은 스타일은 유지하되 다른 HTML 요소로 표현하고자 할 때는 다음 코드 예제와 같이 as 속성을 사용한다. 이 경우 표현되는 HTML 요소 자체가 변경되기 때문에 각각의 요소가 사용할 수 있는 (예제에서 href과 같은) 속성 역시 동일하게 사용할 수 있게된다. (withComponent는 더 이상 사용되지 않음) 자세한 내용은 as polymorphic prop 문서를 참고.

function App() {
  return (
    <Container className="App">
      <Box bgColor="red" />
      <Box bgColor="blue" as="a" href="/" />
    </Container>
  );
}

중첩 스타일링

styled-components는 기본적으로 (마치 SCSS처럼) 다음과 같은 중첩 스타일링을 지원한다.

const Something = styled.div`
  span {
    &:hover {
      background-color: black;
      color: white;
    }
  }
`;

컴포넌트 중첩 스타일링

중첩할 때 HTML 요소나 가상 선택자 외에도, styled-components의 컴포넌트를 중첩 요소로 지정할 수도 있다.

const Span = styled.span``;

const Something = styled.div`
  ${Span} {
    &:hover {
      background-color: black;
      color: white;
    }
  }
`;

이 경우 다음과 같이 as 속성을 사용해서 다른 HTML 요소로 표현해도 지정된 스타일은 유지할 수 있다.

1
2
3

<Something>
  <Span as="p">Hello</Span>
</Something>

Props 전달

다음 코드 예제와 같이 마치 React 컴포넌트에 Props를 전달하는 것과 같은 형식으로 Props를 전달할 수 있다. 이 Props를 통해 같은 스타일 컴포넌트를 사용함과 동시에, 지정한 부분 (예를들어 배경 컬러, 폰트 사이즈 등)만 커스터마이징 해서 재사용할 수 있게된다.

  return (
    <Container className="App">
      <Box bgColor="blue" />
      <Box bgColor="red" />
    </Container>
  );

const Box = styled.div`
  background-color: ${(props) => props.bgColor};
`;

HTML 요소에 속성 값 지정

styled-components로 새로운 컴포넌트를 생성할 때, 다음 코드 예제와 같이 해당 요소의 속성 값을 사전에 설정할 수 있다. 다음 코드 예제에서 Input으로 생성된 input 위젯은 모두 required 속성이 추가된 채 생성된다.

1

const Input = styled.input.attrs({ required: true })``;

테마 사용

styled-components에서 제공하는 ThemeProvider를 사용하면 애플리케이션의 Theme을 지정할 수 있고, 어디서든 해당 Theme이 가지고 있는 설정에 접근할 수 있다.

ThemeProvider

ThemeProvider를 사용하기 위해서는 먼저 다음 코드 예제와 같이 index.js에서 ThemeProvider로 App 컴포넌트를 감싸준다. 이때 ThemeProvider는 theme이라는 속성이 요구된다.

import React from "react";
import ReactDOM from "react-dom";
import App from "./App";
import { ThemeProvider } from "styled-components";

// 테마 파일 예시
const theme = {
  textColor: "black",
  bgColor: "white",
};

ReactDOM.render(
  <React.StrictMode>
    <ThemeProvider theme={theme}>
      <App />
    </ThemeProvider>
  </React.StrictMode>,
  document.getElementById("root")
);

이제 App 컴포넌트는 ThemeProvider 컴포넌트 안에 위치하기 때문에, 다음 코드 예제와 같이 App 컴포넌트 하위 어디에서든 제공되는 Theme의 Props에 접근할 수 있다.

1
2
3
4

const Container = styled.div`
  background-color: ${(props) => props.theme.bgColor};
  color: ${(props) => props.theme.textColor};
`;

import styled, { keyframes } from "styled-components";

const animation = keyframes`
  from {
    transform: rotate(0deg);
  } to {
    transform: rotate(360deg);
  }
`;

const AnimationBox = styled.div`
  width: 100px;
  height: 100px;
  background-color: slategray;
  animation: ${animation} 1s linear infinite;
`;

타입스크립트

타입스크립트 환경에서 styled-components를 사용할 때 Props의 타입을 지정하는 방법이나, Theme을 사용하는 방법에 대해서는 타입스크립트에서 styled-components 사용하기 (Theme, Props) 글을 참고하자.
