Recoil 기본 사용법 요약 정리

Dohae Lee on Oct 15, 20212021-10-15T00:00:00+08:00
react
6 min read

(오늘의 영업 대상) Recoil은 페이스북이 개발하고 관리하는 React 전용 상태 관리 라이브러리이다. 뭐 React의 아버지가 React 전용 상태 관리 라이브러리를 만들어 배포하고 관리하고 있으니 백그라운드는 든든한 편이고, 무엇보다도 배우고 사용하기가 매우 쉽다는 장점을 가지고 있다.

인기도는 NPM Trends 결과를 기준으로 보자면 Zustand와 비슷한 수준. 아무래도 사용자 수로 따지자면 Redux, MobX에 비해 모자란건 (?) 사실이지만, 사용법이 매우 간단하고 직관적이라서 자주 애용하는 친구다.

기본 사용법

먼저 recoil에서 제공하는 RecoilRoot로 애플리케이션을 감싸주기만 하면 Recoil을 사용할 준비는 끝난다.

import React from "react";

import { RecoilRoot } from "recoil";

ReactDOM.render(

이제 Recoil의 상태 단위인 Atoms를 작성한다.

useRecoilValue

해당 상태 값을 꺼내 쓸 때는 recoil에서 제공하는 useRecoilValue() Hook을 사용한다. (참고 : useRecoilValue(state))

1
2
3
4

import { useRecoilValue } from "recoil";
import { isLoggedInAtom } from "./atoms";

const isLoggedIn = useRecoilValue(isLoggedInAtom);

useSetRecoilState

Recoil에서 상태 값을 변경하는 방법은 아주 직관적인데, 값을 꺼내 쓸 때와 마찬가지로 useSetRecoilState() Hook을 사용한다. (참고 : useSetRecoilState(state)) useSetRecoilState()는 Atoms을 인자로 넘기면, Setter 함수를 리턴한다. 이 함수는 useState()가 리턴하는 Setter처럼 생각하면 된다.

1
2

const isLoggedInSetter = useSetRecoilState(isLoggedInAtom);
isLoggedInSetter((current) => !current);

useRecoilState

만약 현재 상태 값과 상태 값을 변경하는 Setter 함수를 한 번에 꺼내 쓰고 싶다면 (마치 useState() 처럼!) useRecoilState()를 사용한다. (참고 : useRecoilState(state))

1

const [isLoggedIn, setIsLoggedIn] = useRecoilState(isLoggedInAtom);

핵심 개념

Recoil의 핵심 개념은 그다지 복잡하지 않다.

Atom

이미 위에서 ‘기본 사용법’을 통해 살짝 맛을 본 Atom은 Recoil의 단위 데이터로, 컴포넌트가 구독할 수 있는 상태 단위를 의미한다. 컴포넌트가 구독하고 있는 Atom (데이터)가 업데이트 될 경우, 해당 컴포넌트는 업데이트 된 데이터를 기반으로 Re-Render 된다. 동작은 Hook처럼 이해하면 쉽다.

1
2
3
4
5
6

import { atom } from "recoil";

export const msgState = atom({
  key: "msg",
  default: "",
});

새로운 Atom을 생성할 때는 위 코드 예제와 같이 recoil에서 제공하는 atom()을 사용하며, atom()은 key와 default 속성을 가진 오브젝트를 인자로 받는다. 이때 key는 Atom을 구별할 텍스트 값을, default는 해당 상태의 기본 값을 의미한다.

Selector

Selector는 Atom의 상태를 기반으로한 동적인 데이터를 파생시킬 때 사용한다. 쉽게 말해, Selector를 사용하면 Atom의 값 (상태)을 가져다가 어떤 가공처리를 해서 새로운 결과를 만들어 낼 수 있다.

import { atom, selector } from "recoil";

export const usersState = atom({
  key: "users",
  default: [],
});

export const usersSelector = selector({
  key: "usersSelector",
  get: ({ get }) => {
    const users = get(usersState);
    return users.length;
  },
});

Selector를 생성하기 위해서는 recoil에서 제공하는 selector()를 사용한다. 기본적인 틀은 atom()을 사용할 때와 동일하나 default가 아니라 get 또는 set (또는 둘 다)가 요구된다. 이때 get은 데이터를 가공하는 함수를, set (Optional)은 가공된 데이터를 저장하기 위해서 RecoilState 객체를 리턴하는 함수를 의미한다.

위 코드 예제에서 usersSelector는 get을 사용하고 있으며, get의 첫 번째 인자로 받는 객체의 get()을 통해 (get(usersState)와 같이) 특정 Atom이나 Selector에 접근할 수 있다. 또한 이때 기본적으로 get()의 인자로 넘긴 Atom이나 Selector와 종속 관계가 되어, 해당 Atom 또는 Selector가 업데이트 되면 해당 Selector도 자동으로 재실행된다.

1
2
3

const users = useRecoilValue(usersState);
const usersSelector = useRecoilValue(usersSelector);
console.log(usersSelector);

따라서 위 코드 예제와 같이 usersSelector를 출력할 때, users의 값이 업데이트 된다면 계속해서 최신의 상태를 기반으로 usersSelector가 재실행되고 가공처리되어 새로운 결과를 리턴하게 된다.
